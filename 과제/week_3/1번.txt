1.연산자 우선순위(Operator precedence)는 수식에서 연산자들이 수행되는 순서를 정하는 규칙입니다. 연산자 우선순위가 높은 연산자는 먼저 수행되며, 우선순위가 같은 연산자는 왼쪽에서 오른쪽으로 수행됩니다. 예를 들어, 곱셈 연산자 ()의 우선순위가 덧셈 연산자 (+)보다 높으므로, 2+34는 2+(3*4)로 계산됩니다.

교환법칙(Commutative property)은 덧셈과 곱셈 연산에서 적용되는 법칙으로, 순서를 바꿔도 결과가 바뀌지 않는 성질입니다. 즉, a + b = b + a, a x b = b x a와 같이 순서를 바꿔도 결과가 같습니다.

결합법칙(Associative property)은 덧셈과 곱셈 연산에서 적용되는 법칙으로, 연산 순서가 바뀌어도 결과가 바뀌지 않는 성질입니다. 즉, (a + b) + c = a + (b + c), (a x b) x c = a x (b x c)와 같이 연산 순서를 바꿔도 결과가 같습니다.
